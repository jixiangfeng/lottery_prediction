## 冻结配置

- 玩法空间：完整 `000-999`，不默认排除豹子、极端和值/跨度或上期原号。
- 输出预算：直选10注、组选10注，排序模式 `ensemble`。
- 频率窗口：`30/50/100/300/1000`，窗口权重 `0.35/0.30/0.20/0.15/0.10`。
- 基础参数：频率权重 `0.7`、遗漏权重 `0.03`、遗漏上限 `30`、多样性权重 `0.3`、边际权重 `1.0`、复合分下限 `2.0`。
- 集成分下限：`0.08`；结构约束为 `soft`，概率下限 `0.02`，惩罚权重 `0.05`。
- 蒙特卡洛：启用，每期 `20000` 次，随机种子为源期号，使用位置对条件和形态接受。
- 逻辑回归：启用，严格历史训练目标 `60` 期，每个正样本配 `9` 个负样本。
- 实盘调权：基础权重如下；逐模型至少100个已开奖前向样本，且按实际候选覆盖率计算的单侧 `p<0.01` 时才按预先固定规则正向调整，单模型最多浮动10%，总权重不变。

## 执行与判定

1. 只接受期号大于 `2026187` 的开奖作为新样本；初始快照只复盘源期之后的第一期开奖。
2. 每次在目标期开奖前，用最新1000期滚动历史生成一次 `--freeze-pick` 快照。下一期开奖入库后，先自动复盘上一快照，再冻结新的下一期快照。
3. 已冻结快照禁止覆盖。参数、依赖或源码指纹变化必须建立新的实验 ID，不能并入 v1 样本。
4. 每期应在开奖前把数据哈希、推荐指纹和候选登记到版本控制或其他可信时间戳系统。本地 `immutable` 只能防止程序误覆盖，不能防止人工改文件。
5. 满500个已开奖前向样本前不宣称算法可行，也不根据阶段结果修改 v1。500期后按固定闸门同时检验直选和组选：单侧 `p<0.01`、相对随机提升至少25%、99% Wilson下界高于随机、三个非重叠时间块均不低于随机。
6. 任一门槛失败，结论均为“不通过”。通过也只代表该前向样本存在统计优势，不保证未来中奖或盈利。

## 后续代码变更说明

概率 v2 开发新增了独立模块和概率字段，因此当前工作树的全源码指纹已不同于实验登记中的初始指纹。`2026187` 初始快照及推荐指纹保持不变；后续若继续生成集成快照，必须登记新的源码指纹和实验 ID，不能并入原 v1 序列。

首次冻结命令：

```bash
make digit-report \
  DIGIT_LOTTERY=fc3d \
  DIGIT_CSV=data/fc3d/official_history.csv \
  DIGIT_FREEZE_PICK=1
```
