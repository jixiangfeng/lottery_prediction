# 三位彩概率 v2 开发评估

## 目标与算法

概率 v2 不再把集成分解释为概率，而是对完整 `000-999` 空间执行以下流程：

1. 精确枚举1000个有序号码，保留全部合法结果。
2. 在 `ensemble` 和14个统计模型剖面中，用严格历史校准段的前2/3选择模型、温度和学习权重。
3. 使用校准段最后1/3作为独立守门段；选参段、守门段或三个时间块任一未改善 Log Loss，实际学习权重立即设为0。
4. 最终分布为 `P=(1-lambda)*Uniform+lambda*Learned`，并重新归一化到概率和为1。
5. 直选直接选择概率Top10，不执行形态配额或多样性替换；概率并列时用源期号哈希确定性打破并列。
6. 组选概率为所有有序排列概率之和，直接选择概率最高的10组，不固定组三/组六配额。

ML负采样概率和蒙特卡洛近似暂不进入v2。前者没有完成总体先验校准，后者在1000状态空间中不如精确枚举。

## 决策

- 不冻结 `fc3d_probability_forward_v2`，因为概率校准闸门失败。
- 不把均匀回退结果包装成预测提升，也不使用同一500期继续筛选更多模型后宣称有效。
- 现阶段没有历史开奖特征可以支持稳定高于均匀随机的概率模型；继续优化应优先寻找独立于开奖号序列的数据机制证据，而不是增加更多同源派生特征。
- 两种彩票并行完整评估约4分钟；后续性能优化方向是缓存逐期统计和概率空间，不改变本次结论。
