# 决策记录

## 2026-07-17 在线概率 v3 采用预测后反馈

- 决策：不再从375组模型/温度/混合权重中逐日择优；固定均匀基线和14个统计专家，每期开奖后按真实号概率做指数权重更新，并向初始先验收缩1%。
- 边界：前500期在线预训练，后500期继续预测后反馈；更新规则在评估前固定，目标期开奖前的预测不读取本期及未来结果。
- 结果：福彩3D和排列三 Log Loss 分别为 `6.912332`、`6.912289`，均差于均匀基线 `6.907755`；最终均匀专家权重都约96.5%。
- 决策：反馈机制保留为独立研究入口，但不接入日报主推荐，不根据同一500期继续调参后重新包装为有效结果。

## 2026-07-17 在线概率 v3 接入独立日报模式

- 决策：新增 `DIGIT_RANKING_MODE=online_probability`，不改变默认 `ensemble`；日报首次重放第101期至当前期，后续仅消费新增开奖。
- 状态：保存配置指纹、历史前缀指纹、已处理期数和专家权重；历史修正、配置变化或状态损坏时自动全量重建。
- 留痕：推荐快照保存在线状态和候选概率，但稳定指纹排除 `cache_hit`/`incremental` 运行元数据，重复运行保持幂等。
- 边界：在线 v3 的500期开发结果未通过随机闸门，因此仅作为可审计研究模式，不接管默认日报。

## 2026-07-17 概率 v2 校准失败时退回均匀分布

- 决策：三位彩精确枚举1000状态；直选按概率纯Top10，组选按排列概率求和，不再用多样性或形态配额改变主评估候选。
- 校准：在 ensemble 和14个统计剖面中用前2/3选择模型、温度、混合权重，后1/3独立守门，并要求三个时间块全部改善 Log Loss。
- 结果：福彩3D选中 `sum`、排列三选中 `latestDistance`，但守门 Log Loss 分别为 `6.940027`、`6.971786`，均差于均匀基线 `6.907755`，因此实际权重设为0。
- 决策：不冻结概率 v2 前向实验；均匀回退用于阻断负向学习信号，不承诺实现样本命中率保底。

## 2026-07-17 冻结福彩3D前向实验 v1

- 决策：以 `2026187` 为数据截止期建立 `fc3d_ensemble_forward_v1`，初始数据哈希、源码指纹、固定参数、推荐指纹和候选写入 `docs/fc3d_forward_protocol.md`。
- 边界：只接受 `2026187` 之后、开奖前已冻结的快照；已经查看的500期留出结果不再用于调参后复测。
- 执行：普通日报快照可覆盖，只有显式 `--freeze-pick` 才不可变；已冻结快照不能被普通运行降级或覆盖。
- 判定：满500个新前向样本前不宣称可行，期满后仍需直选和组选分别通过既定精确随机闸门。

## 2026-07-17 官方数据 500 期验证不通过

- 数据：福彩3D和排列三各取最近 1000 期，前 500 期作为最早目标的训练历史，最后 500 期从 2025-02-18 严格前推至 2026-07-16。
- 福彩3D：集成直选 `15/500` 对随机期望 `5`，`p=0.000206`，但三个时间块命中 `8/6/1`，最近 150 期为 0 次；组选 `29/500` 对期望 `27`，不显著。因此整体不通过。
- 排列三：统计直选 `3/500`、集成直选 `4/500`，随机期望均为 `5`；组选也无显著优势，整体不通过。
- 决策：不根据已查看的 500 期留出结果立即调参并复测；福彩3D信号冻结做未来开奖前验证，排列三维持研究状态。
- 性能：两种彩票并行启用 16 模型、五窗口和 20 次基线约耗时 50 分钟，后续优化优先缓存严格历史训练特征并增加进度输出。

## 2026-07-17 增加显式官方历史抓取

- 决策：日报和回测继续保持纯本地；新增用户显式触发的抓取命令，仅允许福彩官网与中彩网固定白名单。
- 约束：超时、重试、重定向域名校验、期号/号码校验和原子替换全部通过后才更新本地 CSV。
- 范围：当前支持福彩3D和排列三；排列五不引入未经验证的新来源。

## 2026-07-17 预测必须通过精确随机可行性闸门

- 决策：不以单次命中、随机模拟百分位或短窗口领先作为可行证据；直选和组选分别使用每期实际覆盖概率做单侧检验。
- 门槛：至少 500 期、`p<0.01`、相对提升至少 25%、99% Wilson 下界高于随机、三个非重叠时间块均不低于随机。
- 三位彩：直选和组选必须分别通过，避免组选较高的天然命中率掩盖直选失败。
- 结果：报告允许明确输出“不可行”；本次改造不承诺现有策略能够过闸。

## 2026-07-17 三位彩取消默认硬过滤

- 决策：福彩3D和排列三默认保留 `000-999` 完整合法空间，包括豹子、极端和值/跨度和上期原号。
- 原因：这些结果概率较低不等于不可能；未经严格前推证明的硬排除会让部分开奖永久无法命中，并扭曲随机基线。
- 取舍：结构特征继续作为集成排序的软信号；显式硬过滤 API 保留，供独立实验使用。

## 2026-07-17 实盘调权采用同候选空间随机基线

- 决策：模型命中率按每期实际候选数除以完整样本空间比较，不再以 50% 为中心。
- 约束：最低 100 期、精确单侧 `p<0.01` 才允许正向调整，使用加一平滑计算相对随机提升，单模型最多浮动 10% 且总权重不变。

## 2026-07-17 收敛为三种数字彩

- 决策：仅保留福彩3D、排列三、排列五，删除快乐8完整功能树和专用 H5。
- 原因：减少两套架构并存、重型 PyTorch 依赖和玩法无关维护成本。
- 影响：业务流程统一读取本地 CSV；后续增加的官方抓取只能显式执行，不得耦合进日报或回测。
- 兼容：不保留已删除玩法的导入路径或规则代码，调用方应迁移到三种数字彩 API。
