# 项目假设

- 当前预测架构只支持福彩3D和排列三的三位数字空间 `000–999`。
- 排列五不由 learned ranker 支持，CLI 必须明确拒绝。
- 每个目标期只能读取目标期以前的开奖历史。
- `probability_v5`是全新隔离开发挑战器，只写开发报告；不读取、不迁移、不兼容v4状态，不接入日报、研究排名、正式推荐或Frozen。
- v5首版固定四个预注册专家：uniform、EWMA位置边际、EWMA位置对和从头机械更新的旧梯度结构对照；不包含既有证据失败的FTRL。
- v5只用UniformExpert提供均匀兜底，不叠加第二层动态`lambda`；Calibration只从冻结网格选择temperature。
- v5开发协议默认使用500期Search、250期Calibration和500期开发Evaluation，并至少排除最新500期Frozen；历史开发Evaluation不是独立Validation。
- v5允许冻结规则下的EWMA、旧梯度和专家权重机械在线更新，但任何人工修改专家、半衰期、先验权重、学习率或校准网格都必须建立新版本。
- v5的LogLoss/Brier使用未过滤的1000类事前概率，raw Top50和日常策略Top50分别记录；晋级门槛必须使用排除上期原号、豹子最多1个并补足50个的策略口径。
- v5在5000次全流程随机模拟和新的独立500期Validation完成前固定`promotionPassed=false`、研究排名关闭、正式推荐关闭。
- v5完整开发运行前必须只写一次登记协议，锁定数据截止、源码、配置、Frozen边界和执行计划；协议及报告不同内容禁止覆盖。
- v5随机模拟必须完整重放四专家、在线聚合、Calibration和联合闸门；正式模式固定5000次并绑定协议与开发报告，少量smoke永远不能晋级。
- v5随机模拟种子由基础种子和试验编号确定，串行与多进程结果必须逐试验一致；正式运行前仍需完成全长度性能验证。
- v5正式随机模拟必须使用逐试验不可覆盖检查点；恢复运行时协议、参考报告、配置或种子身份不一致必须拒绝。
- Search 用于参数探索，Validation只确认Search选定配置但具有硬否决权；Search或Validation未通过时只写审计报告，不写参数文件，也禁止进入Frozen Test。
- Frozen Test 不得用于特征、窗口、权重、温度、候选预算、目标 profile 或 gate 调整。
- 全历史只作为弱长期先验，近期窗口默认具有更高权重。
- 候选数量是成本；命中率必须与同成本随机基线一起解释。
- `all_hit_only` 只用于 Search/Validation 开发：三个 profile 共用已构建目标特征，分别保存参数和预算曲线。
- 位置池覆盖按每位概率质量固定 Top3/5/7/10 计算，不得使用直选 Top50 中“出现过即覆盖”的宽池。
- 候选预算只由 Search 曲线选择；Validation 只验证预选预算，不能反向选择。
- 多轮开发时，最新 500 期固定为 Frozen Test；此前全部历史按时间切分为 Search 和 Validation，不把 Frozen 段用于调参。
- v4主特征窗口固定为目标期之前最近20/50/150期，默认half-life=50；全量历史主要用于产生更多walk-forward目标期和稳定性验证。
- 相邻目标期使用150期滚动状态；Search目标矩阵以数据指纹、配置和目标索引为键保存为NPZ，允许安全断点恢复。
- 固定基线矩阵必须包含uniform、形态先验/转移、和值跨度和20/50/150/all位置频率。
- direct、group、position使用独立实战状态；任何分项未通过自己的Frozen证据时不得借用其他分项激活。
- 日报写入`reports/daily/<lottery>/`，策略迁移写入`reports/state/strategy_registry.json`。
- 在线自适应开发默认外层连续预测最后500个开发期、每期更新统计、每10期重选参数；每次重选只读取块起点之前最近500期。
- 概率必须支持`P_final=λP_model+(1-λ)P_uniform`；均匀候选必须参与在线Search，`λ=0`表示无信号并强制放弃，不生成研究或正式号码排序。
- 普通训练默认使用`research_calibrated`平滑目标；hit-only只能作为显式对照，不能单独准入。
- 可预测性审计固定使用开发区、499次置换、10期配对块和Benjamini-Hochberg FDR 5%；只有LogLoss/Brier同时改善并通过校正的简单基线才算发现信号。
- 在线梯度实验固定先预测后更新，学习率/λ每10期只由此前300期Search选择、此前100期Validation确认；任何当期结果只能影响下一期及以后。
- 稀疏收缩v4对`position_frequency`使用10倍L2，对和值/数字对/趋势组使用5倍L2，形态特征固定为0；因为设计来自已看过的开发结果，旧开发区输出只能标记为探索性证据。
- 全历史影子状态可用于训练和未来预测，但历史中的任意500期不得重新命名为独立Frozen；若做历史验证，必须扫描全部固定、连续、不重叠的完整500期块并报告所有结果。
- LightGBM挑战模型固定为三位置10分类、521个prior-only特征、3个强正则参数候选和3个均匀收缩候选；外层区块结果只能评价不能反向扩展网格。两个彩种总体及联合门槛失败后，该模型族关闭为负基线。
- Rank-aware FTRL v4.1固定`30/70`概率/Top50边界损失、三个学习率专家、35%单专家上限和最大35%动态λ；历史区块结果只用于一次探索性判断。总体未显著或稳定块不足10/14时禁止继续在同一历史调双目标、FTRL或Hedge参数。
- 特征删除必须经过预测时零化和从头重训两层消融；边界负贡献或单期零化净增不能直接判为有害。当前只有fc3d删除`sum_distribution`形成未过显著性门槛的前瞻影子候选，pl3及共享模型不删除特征。
- 每日v4.1候选只有在历史联合闸门通过、动态λ至少5%且Top50豹子不超过1个时才可进入`userVisibleCandidates`；否则必须放弃，研究排序只留在审计报告。
- 日常锁定影子Top50排除最新一期完全相同号码，按原模型排序最多保留1个豹子并补足固定50个；不得把该组合策略解释为提升单号概率或回测优势。
- 日常预测默认使用确定性中文说明且不调用外部AI；只有显式传入`--ai`并提供被Git忽略的`config/ai.local.json`时才允许访问固定DeepSeek域名。
- AI只润色状态、放弃原因和验证进度，不接收内部`researchTop50`，不得新增、删除、解释或重排候选号码。
- 正式策略未激活时默认不展示号码；用户显式传入`--show-research`后可查看研究Top10及前三名贡献，但`userVisibleCandidates`仍保持为空。
- 组选评估分为直选Top50投影诊断和独立组选Top10；每期基线必须按所选组选的1/3/6个排列数加权，并使用Poisson-binomial而非`K/220`。组选联合闸门失败时不得展示组选候选。
- `behavioral_context_v1/v2/v3/v4`全部封存为历史负证据；行为权重固定为0，不再在同一历史选择特征、半衰期、L2、符号或豹子策略。行为报告只用于复现淘汰结论，不得恢复为日常入口。
- 统一总账按直选Top50固定成本汇总核心v4、LightGBM、FTRL和行为模型，对随机基线p值执行Holm校正；覆盖不足、不同成本或事后单彩种候选必须单列，不能进入可比排名。
- 只有LogLoss、Brier、Top50和时间稳定性全部通过的新独立前瞻证据才允许选择模型；当前`selectedModel=null`、`productionMode=uniform_abstain`，核心只在50/100/200期检查且50期只允许提前淘汰。
- 参数搜索目标预算必须与待锁定预算一致；普通训练默认`direct_objective_top_k=50`，不得用Top10/Top5参数事后宣称Top50/Top3有效。
- Search、Validation、walk-forward和日常输出的直选排序必须共同应用“排除上期原号、豹子最多1个”的固定候选策略；LogLoss/Brier仍基于未过滤的1000候选事前概率。
- 正式Search先固定排名参数，再对排名前三结构穷举temperature和正数λ；校准按LogLoss、Brier、TopK ECE依次裁决，`λ=0`只表示放弃，不参与排名。
- Search和Validation确认均要求至少500期、单侧`p<0.01`、相对随机提升至少25%、99% Wilson下界高于随机、3个时间块全部不低于随机，并要求LogLoss/Brier改善的99%时间块bootstrap下界高于0。
- Search未通过时不得读取Validation；Search通过后必须先原子声明每个玩法/profile独立的一次性Validation锁，再读取Validation。
- 正式搜索空间记录必须列出实际评估的特征配置；局部权重不得超出声明的`[-1.5,1.5]`边界。
- `sum_prob` 是组选概率；`max_perm` 与 `mean_top_perm` 是聚合分数。
- 历史 v1、概率 v2、在线概率 v3 不再提供代码或兼容接口。
- 彩票结果高度随机；历史评估不保证未来命中、中奖或盈利。
- 行为A/B的Top50必须复用日常组合策略；概率损失仍按未过滤的1000候选事前分布计算。行为开发默认排除已消费的最后500期Frozen；未覆盖开发区全部完整500期块、存在放弃期或任一固定块不稳定时禁止晋级。
- 锁定影子状态只允许由源码指纹和内容指纹完全匹配的代码继续增量；源码指纹统一LF行尾，不提供旧状态兼容迁移，任何源码变化都必须完整重训到新路径。
