# 开发假设与约束

1. **玩法范围**
   快乐 8 链路保持不变；数字彩链路维护福彩3D、排列三、排列五的本地 CSV 统计、候选生成、日报与严格逐期前推回测。
2. **Python 3.11 环境**  
   默认在名为 `python311` 的 Conda 环境或等效的 Python 3.11 虚拟环境中执行，确保依赖一致。
3. **数字彩增量快照边界**
   日常福彩3D/排列三/排列五日报默认使用 JSON 增量统计快照；允许对已处理前缀做 O(n) 轻量 SHA-256 摘要校验，但 cache hit/追加路径不得对完整 DataFrame 再调用 `analyze_digit_history(...)`，也不得把 Counter 按样本次数展开。近期状态维护基础窗口 10/30/50/100/300；动态全历史窗口使用独立 `allHistory` 标识，只从全历史聚合重建。
   并发调用在等待锁期间若发现快照已推进，且旧输入是当前快照严格短前缀，则旧调用按其窗口/先验在内存全量构建结果并返回 `stale_view`，但不得持久化；`persisted` 表示返回结果是否由当前快照持久状态代表，`snapshotWritten` 仅表示本次调用是否实际写文件。
4. **历史修正优先正确性**
   快照损坏、历史删减、期号非追加、已处理前缀号码修正或 schema/engine/规则/窗口/先验不匹配时，默认自动全量重建，不使用陈旧统计。进程内 `RLock` 与 macOS/Linux `fcntl.flock` 包住完整事务，原子替换保证跨进程不会半写或由并发旧视图覆盖新快照。
5. **理论概率不是预测**
   理论概率按玩法规则精确枚举等可能号码并缓存；经验统计来自历史开奖。二者分层输出，均不解释为提高中奖率的证据。
6. **前推回测不复用日报快照**
   严格逐期前推必须按每个目标期之前的历史独立计算；日报最新快照只服务当前日常统计，不进入历史截止期计算。
7. **hindsight 回放默认关闭**
   “把当前候选回放全部历史”只保留为显式诊断选项，默认复盘开奖前保存的 prediction snapshot，避免将事后匹配误作预测证据。
3. **离线演示数据有限**  
   仓库附带的 `data/kl8/data.csv` 仅包含少量示例，用于 CI 与快速演示；正式分析需自行下载最新数据。
4. **外部网络访问受限**  
   `src/data_fetcher.py` 只允许访问白名单域名，当前主数据源为中国福彩网 `www.cwl.gov.cn`；旧 500/917500 数据源仅作为兼容配置保留。

5. **H5 用户端数据来源**  
   Vue3 H5 首版作为用户端前端工程，不直连 Python 进程；通过 `scripts/sync_h5_data.py` 将最新 `reports/data/kl8_daily_<期号>.json` 同步为 `h5/public/report-data/latest.json`，前端运行时通过 HTTP 动态读取该 JSON。
6. **分析脚本以 CLI 形式运行**  
   `src/analysis` 下脚本保持命令行入口，导入时如需自定义参数需显式传入或使用子进程调用。
7. **Dirichlet 与规则参数配置**  
   先验强度、方差惩罚与频繁项集阈值默认使用 `config.analysis` 中的值，可通过 CLI 覆盖，但需评估样本量与缓存命中率。
8. **Copula 采样样本量前提**  
   `src/analysis/copula_sampler.py` 估计 80 维相关结构，推荐 `limit_line ≥ analysis.copula.min_draws`（默认 180）；不足时会跳过并记录日志。
9. **图嵌入缓存一致性**  
   期望在分析前执行 `make train-graph`（或直接运行 `scripts/train_graph_embeddings.py`）生成最新嵌入；若缓存缺失，特征通道将自动退化为零向量。
10. **互信息惩罚权重可调**  
   互信息矩阵基于当前窗口估计，如需缩短窗口或切换玩法，应同步调整 `analysis.graph_embedding.weight`，避免惩罚过大或过小。
11. **数字彩前推口径**
   每个目标期只使用期号更早的历史；默认比较 `current_statistics`、`ensemble_voting` 与可复现的 `uniform_random`，三者使用相同过滤条件、候选数和形态配额。
12. **数字彩统计默认值**
   位置频率默认使用 30/50/100/300 期窗口与保守权重，日报再加入当前全历史窗口；概率使用弱贝叶斯先验平滑，遗漏分使用对数压缩并在 30 期封顶。
13. **形态防守配额**
   三位彩默认排除豹子，前 5 注至多 1 注组三且整体以组六为主；排列五“三一一/三二”仅占少量防守配额，不作为主流形态。
# 2026-07-16 数字彩第二轮优化假设

- 候选评分是多窗口、重叠特征加权形成的启发式复合对数分，不是规范联合概率或实际开奖概率；默认 `score_floor=2.0` 表示候选不得低于最高复合模型分 2 个单位，多样性只能在该质量池内选取。
- `frequency_weight`、`random_weight`、旧 `candidates` 字段和 `generate_digit_candidates` API 为兼容保留；第二轮主评分由 `marginal_weight`、`pair_weight`、`shape_weight`、`sum_weight`、`span_weight` 与小权重 `omission_weight` 控制。
- 排列五完整评分由共享的前三位复合评分加后两位边际、相关位置对和五位结构附加分组成；不要求最终候选文本与排列三相同。
- 三位彩组选的过滤空间归一化模型质量按同一无序 key 的有序排列模型权重求和，默认排除豹子并严格按 80% 组六、20% 组三分配；排列五不生成组选。
- 多随机基线默认运行 20 次；单次 `uniform_random` 摘要继续保留用于旧消费方，新增均值、5%/95% 分位与当前策略百分位。
- 嵌套调参只在外层训练集尾部验证固定的 `marginal_only`、`joint_balanced`、`joint_heavy`，不读取外层目标期；历史优于随机也不能推导未来中奖概率。

# 2026-07-16 数字彩集成投票与真实复盘假设

- 集成投票不增加新的“预测概率”，而是把位置、位置对、形态、和值、跨度、奇偶、大小、质合、连号、镜像、和值尾、上期距离、同位置重号、遗漏、蒙特卡洛和机器学习排序共 16 个槽位在同一过滤空间内转换为排名分位，再按固定权重与固定分母融合。未产出结果的蒙特卡洛/ML 槽位为兼容保留，统一使用中性 `0.5` 分位占位；它会改变集成分的绝对尺度，但对候选相对排序不提供信号。
- 日报默认使用 `ensemble` 排序；旧 `composite` 排序继续保留为前推对照与 CLI 可选项。新策略只有在严格前推和真实推荐累计复盘中持续超过同口径随机基线，才可视为存在待进一步验证的历史信号。
- 轻量机器学习使用 sklearn 标准化逻辑回归做候选二分类排序；每个训练目标只使用它之前的历史，并按固定数量抽取负样本。输出只作为排序票，不解释为校准后的真实概率。
- 蒙特卡洛默认日报 20000 次、严格前推每目标期 5000 次；当前实现已在多窗口位置分布上加入位置对条件抽样与形态接受率。排列五或大窗口运行过慢时允许显式关闭，但报告必须记录关闭状态。
- 日报窗口为 30/50/100/300/当前全历史；严格前推可独立比较五种窗口，并仅用目标期之前的数据计算稳定分。
- 数字彩推荐快照不预设“期号 +1”；复盘时选择源期之后实际存在的第一期开奖，避免跨年期号不连续导致错配。已生成快照不得在开奖后改写。

# 2026-07-16 数字彩第三轮闭环完善假设

- 模型级调权只使用开奖前快照的各模型 Top 候选与后续真实开奖；单模型样本少于 5 期保持基础权重，达到门槛后使用加一平滑，单模型权重最多浮动 20%。
- 多窗口遗漏表示“在该窗口可见历史内的当前遗漏”；未出现数字以实际可用窗口样本数截断，不虚构更长遗漏。
- 结构约束默认为 `soft`，仅对多窗口中低于概率下限的奇偶/大小/质合结构施加小权重惩罚；`hard` 需显式启用，避免默认过度筛号。
- 组三和组六的集成分在各自无序候选空间内重新计算模型分位，不将直选集成分直接作为组选分数。
- 联合蒙特卡洛使用边际位置概率、位置对条件概率和形态接受率；它仍是基于历史假设的候选生成器，不是开奖机制模型。

# 2026-07-16 排列五全空间性能与模型接线假设

- 排列五 10 万号码空间的数字、和值、跨度、形态及结构特征属于静态数据，允许用进程内 NumPy 只读缓存复用；每期动态分数、外部模型分数和过滤结果必须按统计对象、配置及外部分数对象隔离。评分池仅保留最近 2 项，以满足同一期生成与 rank 复用并限制长期前推内存；外部分数字典在构造时防御性复制为只读映射。
- 完整过滤空间仍参与复合质量归一化和 16 个模型槽位的排名分位计算；优化只减少 Python 对象构造和重复计算，不缩小候选空间，也不改变严格逐期前推训练边界。
- “可用模型”表示代码支持的 16 个槽位；“实际启用模型”至少包含 14 个统计模型，只有蒙特卡洛或 ML 提供非中性分数并形成模型候选信号时才计入 active。inactive 外部槽位仍以中性 `0.5` 参与固定分母的兼容计算，不等于从数学融合中删除。
- Makefile 的数字彩日报和前推目标默认启用高级模型；直接 walk-forward CLI 保持兼容，必须显式传入 `--advanced-models`。所有模型输出均为排序信号，不解释为实际开奖概率。
