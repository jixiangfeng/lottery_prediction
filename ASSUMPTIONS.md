# 关键假设与权衡

## 项目范围

- 2026-07-17 起项目只保留福彩3D、排列三、排列五。
- “移除快乐8预测相关代码”按完整移除该玩法解释，包含抓取、统计、候选、回测、报告、H5、测试、文档和历史产物，而不只是旧模型脚本。
- 三种数字彩共享规则、数据标准化、统计、候选、报告和前推能力，不保留兼容已删除玩法的接口。

## 数据

- 日报和回测只读取本地 CSV，不会隐式访问网络；用户可显式执行 `make digit-fetch`，从固定白名单的福彩官网和中彩网公开接口抓取福彩3D或排列三。
- 抓取数据写入已忽略的 `data/` 目录，CSV 保留开奖日期和来源 URL；接口结构或数据校验失败时不得写入目标文件。
- 期号必须是纯数字且数值唯一；排序和前推边界使用数值期号。
- 日报允许用 O(n) 前缀摘要发现历史修正，但缓存命中和纯追加不会重算完整统计。

## 统计与候选

- 理论概率来自玩法完整样本空间精确枚举，是数学基线而非预测。
- 候选评分是重叠历史特征的启发式复合分，不是规范联合概率。
- ensemble 固定包含 14 个统计槽位及蒙特卡洛、逻辑回归两个可选槽位。
- 未启用的外部模型使用中性分位保持兼容，不提供候选间相对排序信号。
- 三位彩组选在各自无序数字集合空间独立排序；排列五只提供直选。
- 福彩3D和排列三默认让 `000-999` 全部合法号码进入评分空间，不再按和值、跨度、豹子或上期原号做未经验证的硬排除；调用方仍可显式配置硬过滤。
- 形态、和值、跨度、上期距离等历史特征只参与软排序，不能让合法目标在默认排名空间中消失。
- probability 模式当前只支持三位彩，必须保留完整 `000-999`；直选不做多样性替换，组选概率按全部有序排列精确求和。
- 概率学习分布只有在选参段、独立守门段和三个时间块 Log Loss 全部优于均匀分布时才启用，否则权重固定回退为0。
- 在线概率 v3 固定使用均匀基线和14个统计模型；先预测、后开奖、再按真实号概率更新权重，不在评估段重新搜索温度、学习率或模型清单。
- 在线日报首次消费当前历史的全部已开奖期建立状态；后续只对新增期更新。历史前缀、规则或在线配置指纹失配时自动全量重建，不静默沿用旧权重。

## 验证

- 有效策略证据必须来自严格逐期前推或开奖前推荐留痕。
- 固定“今日候选”回放全部历史默认关闭，不能作为预测效果证据。
- 任何历史领先都不能推导未来中奖概率或盈利能力。
- 随机零假设采用各期实际候选覆盖率：直选为唯一候选数除以完整有序样本空间，三位彩组选按组六 6、组三 3、豹子 1 个有序排列精确计数。
- 策略可行必须同时满足至少 500 个严格前推目标期、精确单侧 `p < 0.01`、相对随机提升至少 25%、99% Wilson 下界高于随机基准、三个非重叠时间块均不低于随机；三位彩直选和组选分别过闸后才整体通过。
- 可行性闸门失败表示当前证据不足，系统必须报告“不可行”，不能用随机基线百分位或局部时间段表现替代。
- 实盘模型权重至少积累 100 期，且按模型每期实际候选数计算的精确单侧 `p < 0.01` 后才允许正向调整；单模型相对基础权重最多浮动 10%。
- 2026-07-17 的官方数据 500 期留出验证已经被查看，后续不得根据该区间调参后再把同一区间称为未见测试集；福彩3D集成直选只能作为待前向复验信号。
- `fc3d_ensemble_forward_v1` 以 `2026187` 为数据截止期，只接受后续开奖前生成并冻结的推荐；满500个新样本前不宣称可行，也不根据阶段结果修改该实验。
- 快照的 `immutable` 只防止程序误覆盖；可信的开奖前时间证明仍依赖在开奖前把数据哈希和推荐指纹登记到版本控制或外部时间戳系统。
- 概率 v2 的历史500期开发评估已经被查看，且校准守门失败；该结果不能通过继续搜索剖面后重新包装成未见验证，也不建立正式前向实验。
- 在线概率 v3 继续使用同一已查看的500期做开发诊断；即使更新规则严格无泄漏，该区间仍不是新的未见验证，参数不得按结果反复调整后宣称有效。
